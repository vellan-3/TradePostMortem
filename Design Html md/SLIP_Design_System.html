Core Tokens

SLIP runs on a narrow, disciplined color system.

The current UI language is built on black surfaces, white type, one electric-blue action color, and fixed product-role accents for Verdict, Mirror, and Payslip.

Canvas

`#000000` base canvas. Almost everything sits on black or near-black surfaces.

Primary Blue

`#3B82F6` is the main action and brand voltage. Use it for CTA emphasis and interactive focus.

Verdict Accent

Red is reserved for risk, caution, and the “before the trade” product lane.

Payslip Accent

Purple is reserved for post-trade diagnosis, review, and reflective analysis.

Typography

Three fonts, three clear jobs.

Syne carries the brand weight.

Use for hero headlines, giant stats, wordmarks, and moments that need visual force.

Instrument Sans keeps the product readable.

Use for section headings, nav labels, CTA labels, card copy, and nearly all functional reading surfaces.

Space Mono marks utility and signal text.

Use sparingly for eyebrows, pills, metadata, scroll hints, and labeled UI states.

Product Roles

The three product lanes are semantic, not decorative.

VERDICT

Before The Trade

Risk scan, token structure, timing position, and pre-entry caution. Red stays tied to this meaning.

MIRROR

During the trade

Live winner comparison, wallet gap analysis, and relative position during active price action.

PAYSLIP

After the trade

Trade autopsy, left-on-table cost, recurring mistake patterns, and post-trade grading.

Spacing

The system breathes through wide section rhythm and dense internal clusters.

Major Rhythm

Use large vertical pacing between scenes: `48`, `64`, `80`, and `144` are the dominant section steps in the current UI language.

Internal Density

Inside cards and grouped content, rely on `12`, `16`, `24`, and `32` before introducing custom offsets. The UI should feel deliberate, not hand-tuned section by section.

Implementation Rules

This is how the current SLIP UI should be implemented.

Use shared tokens Colors, radii, spacing, shadows, and type roles should be centralized. The same visual values should not be redefined ad hoc across sections.
Use shared content maps Nav items, feature stages, stats, and footer groups should come from structured data rather than repeated string fragments.
Keep product semantics fixed Verdict is red, Mirror is blue, Payslip is purple. That mapping carries product meaning and should not drift.
Favor component primitives over giant inline styling The current UI language is strong, but its implementation should become more modular and reusable.
Frame screenshots consistently Mockups should share one card mass, one border logic, one surface treatment, and one internal crop logic.
Preserve the emotional tone SLIP should stay dark, premium, blunt, and product-first. Avoid generic SaaS softness and avoid meme-crypto clutter.
SLIP’s UI system is a dark editorial product language built from `Syne`, `Instrument Sans`, `Space Mono`, a black cinematic canvas, one electric-blue action rail, and three fixed product-lane accents.
